  WARBAND NEXUS - Version 2.5.4 (28.03.2026)
═══════════════════════════════════════════════════════════════════════════

NEW FEATURES
────────────
• Rarity (Mounts only): one-time seed — your WN mount tries + Rarity’s attempts per mount; then import stops (SavedVariables flag) so both addons can run without Rarity’s numbers being re-applied every /reload
• Seed runs only when at least one Rarity mount row has attempts > 0; full behavior in RARITY_IMPORT_README.txt (included in the package)
• Debug (requires /wn debug): /wn raritymountpreview — mapping + counts; /wn rarityseedreset — clear the flag to re-test import

PERFORMANCE
───────────
• To-Do tab: cached uncollected results (mount/pet/toy/title/illusion/achievement) with short TTL to avoid expensive store re-iteration on rapid redraws
• To-Do tab: mount metadata cache with 30s TTL to cut redundant C_MountJournal calls when switching tabs
• To-Do tab: achievement category tree cached for the session (GetCategoryList / GetCategoryInfo once)

BUG FIXES
─────────
• Fixed duplicate notification when a try-counted collectible is obtained (TryCounter + Collection)
• Fixed “Try counter reset” message showing for one-time collectibles (mounts, toys, etc.)
• Fixed To-Do tab stuck loading: CollectionLoadingState resets correctly on achievement scan failure / early exit
• Fixed EnsureCollectionData pipeline stalling when achievement scan fails (completion event always fires)
• Added safety timeout for EnsureCollectionData to avoid a permanent loading overlay
• Fixed To-Do empty categories — UI refresh accepts the “all categories” scan-complete event
• Fixed destructive pruning in mount uncollected logic when the API is temporarily unavailable

IMPROVEMENTS
────────────
• Tooltip: per-drop repeatable flag respected when set explicitly (no override from global index)
• Spectral / Lucent Hawkstrider marked non-repeatable (learn-once dungeon mounts)
• Auto screenshot when a try-counted collectible drops (optional “BAM” moment)
• Try counter chat line: combined “Obtained X after Y tries!” where applicable
• One-time collectibles use the correct message (no bogus “reset” wording)

LOCALIZATION
────────────
• New / updated locale keys for 2.5.4 What’s New (enUS); other locales fall back to prior changelog strings until translated
