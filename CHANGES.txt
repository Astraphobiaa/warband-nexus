Warband Nexus 2.5.6

Improvements
- Realm display: Blizzard’s normalized realm strings (e.g. TwistingNether) are shown with friendlier spacing in more UI — Gear character selector, weekly plan headers, Plans tracker cards, and the tracking confirmation dialog. Storage and character keys still use the API/normalized form for consistency.
- Collectible source database: expanded NPC list for Reins of a Tamed Bloodfeaster (BfA Nazmir) and clearer file comments for raid chest vs boss rows, intentional duplicate mount entries, and multi-corpse / AoE loot behavior for try counting.
- Character tooling: `Utilities:SplitCharacterKey` and safer parsing anywhere `Name-Realm` is derived from a string (first hyphen only, full realm suffix preserved).
- Contributor workflow: CONTRIBUTING.md updates (collectible audits, external mount DB file layout); optional `scripts/extract_external_db_npcs.py` and `scripts/audit_external_npcs_vs_wn.py` for NPC / mount-source checks (clone the reference addon repo locally; path is gitignored).

Bug fixes
- Try Counter: merging chat-loot NPC mappings no longer clears a mount’s CHAT path when several NPCs share the same drop table — fixes missing attempts for mounts such as Tamed Bloodfeaster when loot is reported via chat with multiple possible sources.
- Hyphenated realm names (e.g. Azjol-Nerub): `GetCanonicalCharacterKey`, `GetAllCharacters` key repair, Statistics UI, and minimap label parsing no longer treat the wrong hyphen as the name/realm boundary. One-time migration `_realmSuffixRepairV1` repairs truncated `realm` fields from older saves before key normalization runs.
- Collectible DB item IDs: Verdant Skitterfly (192764, was 192055 on Expedition Scout’s Pack); Red Qiraji Resonating Crystal (21321, was incorrectly 21221).
- Character tracking dialog: safer `GetRealmName` handling with secret-value guard where applicable.

Localization
- All supported locales include `TRY_COUNT_CLICK_HINT` and `TRY_COUNT_RIGHT_CLICK_HINT` (try-count editor tooltips).

Repository / packaging
- Removed redundant one-off scripts and scratch files; one-time legacy mount try-import behavior is documented in CONTRIBUTING instead of a separate package readme.
